# ExamSentinel

ExamSentinel is a secure online examination system that enforces exam integrity at the operating system level. Unlike browser-based proctoring tools that can be trivially bypassed by switching applications or running inside a virtual machine, ExamSentinel is a native Windows desktop application that operates with administrator privileges to directly control the testing environment.

The system addresses a fundamental gap in online examination security. Students taking exams through web browsers can alt-tab to search engines, use remote desktop tools to receive external help, copy-paste from reference material, or simply run the entire browser inside a virtual machine where all monitoring becomes useless. ExamSentinel eliminates these attack vectors by implementing OS-level lockdown that blocks keyboard shortcuts (Alt+Tab, Win key, PrintScreen), enforces fullscreen mode, monitors window focus, scrubs the clipboard, kills blacklisted processes (screen recorders, remote desktop tools), detects multiple monitors, and restricts mouse movement outside the exam window. Every violation is timestamped and reported to the server in real-time for teacher review.

Beyond lockdown, ExamSentinel implements a two-stage virtual machine detection system. The first stage (Standard Detection) checks for obvious VM indicators: guest tool processes, registry keys, WMI hardware strings, and MAC address vendor prefixes. These checks catch default VM installations but can be bypassed by a knowledgeable student who removes guest additions and spoofs system identifiers. The second stage (Stealth Detection) uses hardware-level signals that cannot be faked from user-mode: RDTSC timing measures the CPU cycle cost of CPUID instruction execution, which is 50-150 cycles on bare metal but 3,000-50,000+ cycles on any hypervisor due to mandatory VM-exit traps; thermal zone queries detect the absence of physical temperature sensors that every real PC exposes; and SCSI disk identifier checks read virtual disk controller names from the hardware registry layer, which persist regardless of guest tool removal or BIOS string spoofing.

The architecture consists of a Python/Tkinter desktop client distributed as a standalone Windows executable, and a Flask REST API backend with PostgreSQL deployed on Railway. Teachers create courses, enroll students, author MCQ exams, and review session incidents. Students launch the application, pass the pre-exam integrity checks, take their exam under full OS lockdown, and submit. The teacher receives a complete timeline of every lockdown violation that occurred during each student's session.
